### Signal: Curriculum Data Lock-in Confirmed — Market Leaders Store Degrees as Unstructured HTML
- **Source:** GitHub coursedog-importer repo (AustinMLove/coursedog-importer, 9 PRs across March–April 2026) — documents a real university migration from Modern Campus → Coursedog
- **Score:** 9
- **Type:** competitor_weakness / market_gap
- **Detail:** Both Coursedog and Modern Campus store degree requirements as freeform HTML blocks, NOT structured data. Coursedog's `ProgramUpdateRequest` model uses `requisitesFreeform` — a single HTML string that concatenates all requirement sections. Modern Campus's HTML is scraped from catalog pages by anchoring between `<h2>Degree Requirements</h2>` and `<h2>Completion Plan</h2>`, with CSS classes (`li.li2` = course entries, `li.li5` = narrative text) as the only semantic markers. Neither platform supports structured curriculum data that could be programmatically analyzed or assessed. This is the STATUS QUO in the market — a fundamental architectural gap.
- **Why DFVA:** DFVA's core value proposition (scoring/assessing degree programs) is IMPOSSIBLE with unstructured HTML data. The entire curriculum management market stores degrees as formatted text, not as analyzable data. This validates DFVA's structured assessment model as a genuine differentiator — it's not just better analytics, it's a different data model entirely. The market needs a tool that understands curriculum as structured, scorable objects, not HTML paragraphs.
